Hacking the Library: Community Calendar Spring 2017
I worked with another student to build a community calendar, which is now hosted on Heroku. The code is a standard flask webapp. We have a server.py file that acts as the skeleton of the website, some static resources like backgrounds and CSS files, and html templates that display the data to the user. Data is passed from HTML templates to python by HTML form inputs. Flask receives this as a dictionary request.form, and parses it to retrieve the relevant information. Data is passed back into HTML by way of Flask's render_template method. The events themselves are stored in a postgreSQL server provided by Heroku (hobby-dev level, 10000 rows available - if necessary, we could purchase a more expensive database plan). We use psycopg2 as a connector between the python code and the database.

Hacking the Library: Calmmute Spring 2017
Calmmute is built upon the idea that commutes are cognitively important: having time between engagements allows our brains to finish processing the last thing we did before it begins work on the next. Working with another student, I tried to provide a space of sorts to help people "calmmute." There are two pages of our site: The greeting page and the Calmmute page. The greeting page is built in index.html and server.py, and is made up of html and some javascript/jquery. The Calmmute page is built in calmmute.html, pose.html, and server.py, and is made up of html, javascript/jquery, and embedded elements from The Meditation Podcast's soundcloud page and www.yoga.com for the music and yoga pose respectively.

Principles of Engineering: Line-Following Robot Fall 2016
In this lab, I worked with a partner and built a line-following robot, using an Arduino UNO, a motor shield, and two infrared reflectance sensors. We used the IR sensors to detect the presence of black tape, which formed a track on the ground for the robot to follow. Our controller logic used sensor information to compute the signals that must be sent to the motors to enable the robot to follow the line. Our controller uses bang-bang control, in which only two discrete commands are used; one command speeds up a motor and slows down the other, while the other performs the same action on the opposite motors.

Software Design: Harry Potter and the Biwizard Tournament Spring 2016
In partnership with one other student, I contributed to the creation of this Harry Potter-inspired dueling game, which was built using Python. In this game, the player battles against Voldemort and his legion of villains. Each enemy casts a number of spells, which inflict damage on the player. The player can return fire by waving their own wand in certain patterns, thereby casting spells. The game uses a model-view-controller organizational system, in which the mode contains the state of core gameplay mechanics (including hitpoints and spell-casting statuses), the view displays the game itself (using the pygame and OpenCV libraries), and the controller allows the player to use a "wand" (which is detected through OpenCV) to play the game.

Software Design: Text Mining Spring 2016
In this project, I analyzed text that I mined from Project Gutenberg's The Complete Works of William Shakespeare and generated poems based on the frequency of words that appeared in Shakespeare's texts using Python. For example, articles like 'the' and 'a' appear quite frequently. Pronouns are second-most common, followed by more common nouns, more common verbs, and so on. In addition, I've always liked the doge meme, so I modeled my poems after the syntax of the meme. That is to say, many of the poems generated make very little sense, but they're pretty fun to read.

Mechanical Systems: Spinning Tops Fall 2015
I worked in a team of 2 and focused on modeling the motion of mechanical systems using MATLAB. The mathematical models created involved systems of differential equations. The system my partner and I chose, a spinning top, was mathematically complex, in that it involved vector quantities instead of scalar quantities, and second order differential equations instead of first order. A spinning top rotates in all three planes, further complicating the model. In order to model the motion of the top, my partner and I took a crash course in linear algebra and implemented rotational matrices.




Thermal Systems: Cubical Solar Oven Fall 2015
Working with a partner, I simulated the transfer of heat from the environment to a cubical solar box oven with varying dimensions and compositions using MATLAB. The goal was to optimize a cubical solar oven. Specifically, we wanted to determine the dimensions and the composition that would allow a cubical solar oven to heat up as quickly as possible, given near-constant sunlight exposure.
